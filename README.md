# cap

Generate commit messages with AI. Builds to a native executable with encrypted credentials.

## Install

```bash
bun install
bun run setup          # Interactive setup: choose provider, model, enter API key
bun run build          # Compile to native binary 'cap'
bun run install:global # Install to ~/.local/bin
```

## Usage

```bash
git add .
cap
```

## How It Works

**Setup** (`bun run setup`):
- Lists available AI providers and models from `pi-ai`
- Fuzzy-search to filter providers/models
- Encrypts API key with AES-256-GCM
- Saves to `src/encrypted-config.ts`

**Build** (`bun run build`):
- Compiles TypeScript to native executable using `bun --compile`
- Embeds encrypted credentials in the binary
- Output: `cap` (~62MB, no runtime needed)

**Runtime**:
- Reads staged git changes (diff, files, commit history)
- Sends to AI with prompt requesting conventional commit format
- Parses response into subject + body
- Runs `git commit` with generated message

## File Structure

```
src/
  index.ts              # Main CLI logic
  encrypted-config.ts   # Encrypted credentials (generated by setup, gitignored)
scripts/
  build-encrypt.ts      # Interactive setup script
```

## Uninstall

```bash
rm ~/.local/bin/cap
```
